一. 创建用户:
命令:CREATE USER ‘username’@’host’ IDENTIFIED BY ‘password’;

例子:
CREATE USER ‘newpay’@’localhost’ IDENTIFIED BY ‘newpay’;
CREATE USER ‘newpay’@’%’ IDENTIFIED BY ‘newpay’;
CREATE USER ‘ad’@’localhost’ IDENTIFIED BY ‘newpay’;
CREATE USER ‘ad’@’%’ IDENTIFIED BY ‘newpay’;

CREATE USER ‘test’@’localhost’ IDENTIFIED BY ‘test’;
CREATE USER ‘test’@’%’ IDENTIFIED BY ‘test’;
PS：username - 你将创建的用户名,
host - 指定该用户在哪个主机上可以登陆，此处的”localhost”，是指该用户只能在本地登录，不能在另外一台机器上远程登录，如果想远程登录的话，将”localhost”改为”%”，表示在任何一台电脑上都可以登录;也可以指定某台机器可以远程登录;
password - 该用户的登陆密码,密码可以为空,如果为空则该用户可以不需要密码登陆服务器。

新建数据库
CREATE DATABASE IF NOT EXISTS newpay DEFAULT CHARSET utf8 COLLATE utf8_general_ci;

二.授权:
命令:GRANT privileges ON databasename.tablename TO ‘username’@’host’

PS: privileges - 用户的操作权限,如SELECT , INSERT , UPDATE 等(详细列表见该文最后面).如果要授予所的权限则使用ALL.;databasename - 数据库名,tablename-表名,如果要授予该用户对所有数据库和表的相应操作权限则可用表示, 如.*.

例子:
GRANT SELECT, INSERT ON mq.* TO ‘dog’@’localhost’;
grant all privileges on newpay.* to newpay@’localhos’ identified by ‘newpay’;
grant all privileges on newpay.* to newpay@’%’ identified by ‘newpay’;
flush privileges;

PS:必须执行flush privileges; 

GRANT ALL PRIVILEGES ON *.* TO 'newpay'@'192.168.1.170' IDENTIFIED BY 'newpay' WITH GRANT OPTION;
GRANT ALL PRIVILEGES ON *.* TO 'ad'@'%' IDENTIFIED BY 'newpay' WITH GRANT OPTION;
GRANT ALL PRIVILEGES ON *.* TO 'ad'@'localhost' IDENTIFIED BY 'newpay' WITH GRANT OPTION;

FLUSH PRIVILEGES

四.设置与更改用户密码
命令:SET PASSWORD FOR ‘username’@’host’ = PASSWORD(‘newpassword’);

例子: SET PASSWORD FOR ‘newpay’@’localhost’ = PASSWORD(“newpay-new”);
SET PASSWORD FOR ‘test’@’localhost’ = PASSWORD(“test”);
SET PASSWORD FOR ‘test’@’%’ = PASSWORD(“test”);
五.撤销用户权限
命令: REVOKE privilege ON databasename.tablename FROM ‘username’@’host’;

  说明: privilege, databasename, tablename - 同授权部分.

  例子: REVOKE SELECT ON mq.* FROM 'dog2'@'localhost';

PS: 假如你在给用户‘dog’@’localhost’’授权的时候是这样的(或类似的):GRANT SELECT ON test.user TO ‘dog’@’localhost’, 则在使用REVOKE SELECT ON . FROM ‘dog’@’localhost’;命令并不能撤销该用户对test数据库中user表的SELECT 操作.相反,如果授权使用的是GRANT SELECT ON . TO ‘dog’@’localhost’;则REVOKE SELECT ON test.user FROM ‘dog’@’localhost’;命令也不能撤销该用户对test数据库中user表的Select 权限.

  具体信息可以用命令SHOW GRANTS FOR 'dog'@'localhost'; 查看.

六.删除用户
命令: DROP USER ‘username’@’host’;
